# Prj-job 修改说明

## 本次已修改

1. 权限管理弹窗
   - 删除右侧“生效权限预览”区域。
   - 权限管理弹窗改为两列布局：左侧账号信息 220px，右侧权限选择自适应。
   - 权限选择项统一为“权限名称 + 权限代码 + 允许 + 禁用”的行式排布。
   - 允许/禁用按钮固定 88px，减少挤压和错位。

2. 部门管理弹窗
   - 部门管理不再复用 720px 的账号弹窗尺寸。
   - 新增独立布局：弹窗宽度 1040px，左侧表单/权限 650px，右侧部门列表 320px。
   - 部门权限选择项改为和权限管理一致的行式排布。
   - 窄屏下自动变为单列，避免横向挤压。

3. 任务详情弹窗
   - 删除管理总览内重复渲染的任务详情弹窗。
   - 现在任务详情弹窗只由统一入口 public/js/render.js 渲染。
   - 修复打开弹窗后右上角 × 无法关闭的问题。
   - 修复留言提交后可能刷新/回到管理总览的问题。

4. 留言表单
   - 修正发送按钮选择器：从第一个 button 改为 button[type="submit"]。
   - 留言成功后保留当前任务 ID，并保持任务详情弹窗打开。

5. 附件列表
   - 删除重复的 renderFiles 函数。
   - 修复 canDownload 未定义导致附件列表可能报错的问题。

6. 冗余旧代码清理
   - 删除 renderPermissionModalLegacy。
   - 合并 overview-page.js 中重复定义的 renderOverviewPage、bindOverviewEvents、renderOverviewExpandedPanel、renderServiceLoadCard 等函数。
   - 删除 task-detail.js 中重复的 renderInlineInfo，以及不再使用的 renderBriefEditor、renderStatusEditor。
   - 清理 CSS 中已不再使用的 permission-editor、permission-preview、permission-chip-list 等旧样式。

7. 权限数据保存
   - 前端保存权限时会去重，并避免同一个权限同时出现在“允许”和“禁用”。
   - 后端账号权限和部门权限接口也增加了去重与冲突清理。
